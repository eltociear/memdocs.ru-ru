# <a name="group-policy-settings-for-desktop-analytics"></a>Параметры групповой политики для Аналитики компьютеров

*Область применения: Configuration Manager (Current Branch)*

В этой статье описываются параметры локальной и групповой политик в Windows, используемые Configuration Manager и Аналитикой компьютеров.

Когда Configuration Manager регистрирует устройства в Аналитике компьютеров, он устанавливает политики Windows для настройки устройства. В большинстве случаев используйте Configuration Manager только для настройки этих параметров.

## <a name="windows-settings"></a>Параметры Windows

Configuration Manager устанавливает политики Windows в одном или обоих из следующих разделов реестра:

- Объект групповой политики (**GPO**): `HKLM:\SOFTWARE\Policies\Microsoft\Windows\DataCollection`

- Предпочтение **локальной политики**: `HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\DataCollection`

| Policy (Политика) | Путь | Область применения | Значение |
|--------|------|------------|-------|
| **CommercialId** | Локальная | Все версии Windows | Чтобы устройство отображалось в Аналитике компьютеров, настройте его с помощью коммерческого идентификатора вашей организации. |
| **AllowTelemetry**  | GPO | быть под управлением ОС Windows 10; | Установлено `1` для **базовых**, `2` для **расширенных** или `3` для **полных** диагностических данных. Для Аналитики компьютеров требуются как минимум базовые диагностические данные. Корпорация Майкрософт рекомендует использовать уровень "Расширенный (ограниченный)" с Аналитикой компьютеров. Дополнительные сведения см. в статье [Настройка диагностических сведений Windows в вашей организации](https://docs.microsoft.com/windows/configuration/configure-windows-diagnostic-data-in-your-organization). |
| **LimitEnhancedDiagnosticDataWindowsAnalytics** | GPO | Windows 10 версии 1803 и более поздних версий | Этот параметр применяется, только если для параметра AllowTelemetry установлено значение `2`. Он ограничивает события расширенных диагностических данных, отправляемых в Майкрософт, только теми событиями, которые необходимы для Аналитики компьютеров. Дополнительные сведения см. в статье [Расширенные события и поля диагностических данных Windows 10, используемые Windows Analytics](https://docs.microsoft.com/windows/configuration/enhanced-diagnostic-data-windows-analytics-events-and-fields). |
| **AllowDeviceNameInTelemetry** | GPO | Windows 10 версии 1803 и более поздних версий | Разрешите устройствам передавать имя устройства. Имя устройства не отправляется в корпорацию Майкрософт по умолчанию. Если вы не отправите имя устройства, оно отобразится в Аналитике компьютеров как "Неизвестно". Дополнительные сведения см. в разделе [Имя устройства](enroll-devices.md#device-name). |
| **CommercialDataOptIn** | Локальная | Windows 8.1 и более ранние версии | Для Аналитики компьютеров требуется значение `1`. Дополнительные сведения см. в статье [Windows Customer Experience Improvement Program, Commercial Data Opt-in and Resulting Internet Communication in Windows 7 and Windows Server 2008 R2](https://docs.microsoft.com/previous-versions/windows/it-pro/windows-7/ee126127\(v=ws.10\)) (Программа улучшения качества ПО Windows, согласие на сбор коммерческих данных Windows и выход в Интернет в Windows 7 и Windows Server 2008 R2). |
| **RequestAllAppraiserVersions** | Оба варианта | Windows 8.1 и более ранние версии | Для правильной работы сбора данных Аналитики компьютеров требуется значение `1`. |
| **DisableEnterpriseAuthProxy** | GPO | Все версии Windows | Если в вашей среде для доступа к Интернету требуется прокси-сервер с встроенной проверкой подлинности пользователей Windows, то для правильной работы сбора данных Аналитики компьютеров требуется значение `0`. Дополнительные сведения см. в разделе [Проверка подлинности прокси-сервера](enable-data-sharing.md#proxy-server-authentication). |

> [!IMPORTANT]
> В большинстве случаев используйте Configuration Manager только для настройки этих параметров. Не применяйте также эти параметры в объектах групповой политики домена. Дополнительные сведения см. в разделе [Разрешение конфликтов](enroll-devices.md#conflict-resolution).

### <a name="settings-from-upgrade-readiness"></a>Параметры Проверки готовности к обновлению

Windows Analytics также настраивает следующие политики с помощью скрипта Проверки готовности к обновлению:

- CommercialId
- AllowDeviceNameInTelemetry
- CommercialDataOptIn
- RequestAllAppraiserVersions

Если вы запускали скрипт подключения Проверка готовности к обновлению на устройстве, эти параметры политики могут все еще существовать. Не используйте устаревший скрипт. Перед регистрацией устройства в Аналитике компьютеров удалите предыдущие параметры политики.

## <a name="group-policy-settings"></a>Параметры групповой политики

В основном используйте коллекции Configuration Manager для настройки параметров и регистрации Аналитики компьютеров. Используйте непосредственное членство или запросы для включения устройств в коллекцию или исключения из нее. Дополнительные сведения см. в статье [Создание коллекций](../core/clients/manage/collections/create-collections.md).

Configuration Manager настраивает коммерческий идентификатор и параметры диагностических данных для целевой коллекции. Если необходимо настроить разные параметры диагностических данных для разных групп устройств, используйте параметры групповой политики для переопределения параметров Configuration Manager. Например, вам необходимо установить уровень **Расширенные (ограничено)** для некоторых устройств и **Базовый** для других. У некоторых устройств могут быть другие параметры [проверки подлинности прокси-сервера](enable-data-sharing.md#proxy-server-authentication).

Соответствующие параметры групповой политики находятся по следующему пути: **Конфигурация компьютера** > **Административные шаблоны** > **Компоненты Windows** > **Сборки для сбора данных и предварительные сборки**.

Параметры групповой политики изменяют параметры реестра только в следующем разделе: `HKLM:\SOFTWARE\Policies\Microsoft\Windows\DataCollection`

> [!IMPORTANT]
> При использовании параметров групповой политики для реализации сложных сценариев уделите особое внимание параметрам политики, которые могут вызывать конфликты конфигурации. Configuration Manager настраивает [параметры Windows](#windows-settings), только *если значение еще не существует*. Параметры групповой политики имеют приоритет над параметрами Configuration Manager, поэтому некоторые конфигурации групповой политики могут вызвать проблемы с Аналитикой компьютеров.

### <a name="group-policy-settings-that-could-conflict-with-configuration-manager-settings-for-desktop-analytics"></a>Параметры групповой политики, которые могут конфликтовать с параметрами Configuration Manager для Аналитики компьютеров

Параметры групповой политики, приведенные в таблице ниже, с наибольшей вероятностью могут конфликтовать с параметрами Windows, которые Configuration Manager задает на устройствах, регистрируемых в Аналитике компьютеров.

| Отображаемое имя | Значение реестра | Действие на устройства, зарегистрированные в Аналитике компьютеров |
|--------------|----------------|-------------------------------------------------|
| **Настроить коммерческий идентификатор** | CommercialId | Если задать для этой политики другое значение, оно переопределяет коммерческий идентификатор, установленный Configuration Manager. Если этот идентификатор отличается, настроенные устройства могут не отображаться в Аналитике компьютеров. |
| **Разрешить телеметрию** | AllowTelemetry | Если задать для этой политики другое значение, оно переопределяет глобальный уровень диагностических данных, установленный в Configuration Manager для целевой коллекции. |
| **Ограничить расширенные диагностические данные до минимума, требуемого Аналитикой компьютеров** | LimitEnhancedDiagnosticDataWindowsAnalytics | Эта политика зависит от предыдущего параметра AllowTelemetry. В зависимости от уровня, заданного в Configuration Manager или с помощью групповой политики, эта политика может изменить уровень диагностических данных на устройстве на **Расширенные** или **Расширенные (ограничено)** . Эта политика применяется, только если параметр AllowTelemetry имеет значение `2` (**Расширенные**). |
| **Разрешить отправку имени устройства в диагностических данных Windows** | AllowDeviceNameInTelemetry | Если вы согласились на отправку имен устройств в Configuration Manager, эту настройку можно переопределить, задав для данной политики значение "Отключено". При отключении этого параметра вместо имен устройств в Аналитике компьютеров отображается значение "Неизвестно". Дополнительные сведения см. в разделе [Имя устройства](enroll-devices.md#device-name). |
| **Настройка использования проверенного прокси-сервера для службы "Функциональные возможности для подключенных пользователей и телеметрия"** | DisableEnterpriseAuthProxy | Если настроить для устройств Configuration Manager использование прокси-сервера с проверкой подлинности пользователей (`0`), а затем задать для этой политики значение **Отключить использование проверенного прокси-сервера** (`1`), устройство будет отправлять диагностические данные в контексте системы, а не в контексте пользователя. Если для устройства не настроен прокси-сервер в контексте системы или устройство не может пройти проверку подлинности на прокси-сервере, Windows не сможет отправлять диагностические данные в Аналитику компьютеров. |

> [!NOTE]
> Устаревшая политика **Настройка телеметрии и функциональных возможностей подключенных пользователей** (TelemetryProxy) позволяет Windows пересылать диагностические данные на выделенный прокси-сервер вместо использования прокси-сервера пользователя (WinINET) или устройства (WinHTTP). Некоторые компоненты Windows не поддерживают эту политику. Если вы используете эту политику, она может вызывать проблемы с качеством данных в Аналитике компьютеров.

#### <a name="behavior-of-disabled-settings"></a>Поведение отключенных параметров

Присвоение этим параметрам групповой политики значения **Отключено** по-разному влияет на работу системы.

- При отключении политики **CommercialId** система Windows удаляет значение реестра. Затем параметр Configuration Manager для коммерческого идентификатора, который задается в пути к локальной политике в реестре, применяется к устройству.

- Для политик, которые Configuration Manager задает в том же расположении реестра, что и групповая политика, при отключении параметра в групповой политике Windows удаляет значение реестра. Configuration Manager снова задаст его в следующем цикле обработки политики, после чего Windows удалит его при следующем обновлении групповой политики. Это постоянное изменение конфигурации может привести к нежелательному поведению Аналитики компьютеров.

  - Если присвоить этим параметрам групповой политики значение **Не настроено**, Windows удалит это значение один раз, но больше не будет удалять его. Эта конфигурация позволяет Configuration Manager применять значения ожидаемым образом.

### <a name="group-policy-settings-to-customize-the-user-experience"></a>Параметры групповой политики для настройки взаимодействия пользователя с данными

Эти параметры групповой политики не являются обязательными для Configuration Manager или Аналитики компьютеров. Их можно задать в групповой политике, чтобы настроить взаимодействие пользователей с диагностическими данными Windows.

| Отображаемое имя | Значение реестра | Действие на устройства, зарегистрированные в Аналитике компьютеров |
|--------------|----------------|-------------------------------------------------|
| **Настроить уведомления об изменении согласия на телеметрию** | DisableTelemetryOptInChangeNotification | Начиная с версии 1803 Windows 10 система уведомляет пользователей об изменении уровня диагностических данных. Используйте эту политику, чтобы отключить уведомления. |
| **Настроить пользовательский интерфейс для параметра сбора данных телеметрии** | DisableTelemetryOptInSettingsUx | При настройке уровня диагностических данных вы устанавливаете верхнюю границу для устройства. Начиная с версии 1803 Windows 10 пользователи могут устанавливать более низкий уровень. Используйте эту политику, чтобы запретить пользователям изменять уровень диагностики. Дополнительные сведения см. в статье [Настройка диагностических сведений Windows в вашей организации](https://docs.microsoft.com/windows/privacy/configure-windows-diagnostic-data-in-your-organization#enterprise-management). |
| **Отключить удаление диагностических данных** | DisableDeviceDelete | Начиная с версии 1809 Windows 10 пользователи могут удалять диагностические данные на странице параметров **Отзывы и диагностика**. Используйте эту политику, чтобы запретить удаление диагностических данных, собранных корпорацией Майкрософт с устройства. |
| **Отключить Просмотр диагностических данных** | DisableDiagnosticDataViewer | Начиная с версии 1809 Windows 10 пользователи могут включать и открывать средство просмотра диагностических данных со страницы параметров **Отзывы и диагностика**. Используйте эту политику, чтобы отключить средство просмотра диагностических данных в параметрах Windows и запретить отображение в нем диагностических данных, собранных корпорацией Майкрософт с устройства.|
